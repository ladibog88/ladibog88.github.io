🧩 The Problem

As someone on a year-long sabbatical juggling career development, language learning, personal reflection, and constant curiosity — my thoughts lived everywhere and nowhere. Ideas in Telegram messages to myself. Resources buried in browser tabs. Thai vocabulary disappearing into the notes app. To-dos spread across three different tools.

I had no single place where everything lived — and no way to look back at how my thinking had evolved over time. I needed a second brain. So I built one.

⚙️ How It Works

📲
Send anything to the Telegram bot A thought, link, photo, Thai word, to-do — whatever's on your mind
🗂
Choose a category with one tap 6 categories appear as inline buttons — no typing needed
⚡
Set priority + due date (To-Dos only) High / Medium / Low, then type a date or skip
🏷
Add an optional keyword tag e.g. "python", "thai", "career" — for filtering later
✅
Saved to Google Sheets forever With timestamp, category, priority, content, and keyword
📊
View anytime in the Streamlit dashboard Filter, search, visualise, and mark things done

📊 The Dashboard

The Streamlit dashboard has four pages, each designed around how I actually review my captures:


📋 Main Table

Every entry filterable by category, keyword, and status. Mark anything done with one click — completed entries strike through and collapse.

✅ To-Do Tracker

Tasks grouped into 🚨 Overdue, 🔴 High, 🟡 Medium, 🟢 Low. Due dates calculated dynamically — nothing gets lost.

🌏 Language Log

A clean vocab list for Thai and Spanish. Filterable by language, searchable by word. Displays country flags automatically.

📊 Breakdown

Category donut chart, weekly activity bar chart, and top keywords ranked — a visual map of where your mind has been.

🧠 What I Learned

🔐

Secrets Management — The Hard Way

Accidentally pushed API keys to a public GitHub repo early on. Immediately revoked everything, scrubbed the git history, and rebuilt security practices from scratch. A painful but unforgettable lesson in .gitignore and environment variables.

⚙️

Async Python

Telegram bots run on asyncio under the hood. Debugging async errors — especially Python 3.14 compatibility issues — taught me how the event loop works and why version pinning matters.

🌐

Multi-Environment Deployment

Getting identical code to run on Windows locally, Railway (Linux), and Streamlit Cloud (Linux) required understanding how each platform handles Python versions, dependencies, and secrets differently.

🏗

Building in Phases

Starting with raw capture only — no AI, no fancy features — meant I had something working and usable from day one. Each phase built on a stable foundation rather than trying to build everything at once.

🔌

API Limitations Are Real

Google service accounts can't upload to Drive due to storage quota restrictions. Discovered this the hard way and pivoted to Cloudinary — a good lesson in reading API documentation before building.
